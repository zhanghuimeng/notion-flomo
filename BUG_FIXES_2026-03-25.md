## 🐛 BUG 详情

### 1. 环境变量解析 BUG

**严重程度**: 🔴 高（导致功能完全失效）

**问题描述**:
```python
# flomo2notion.py:352 (修复前)
full_update = os.getenv("FULL_UPDATE", False)
interval_day = os.getenv("UPDATE_INTERVAL_DAY", 7)

if not full_update and not is_within_n_days(...):
    continue  # 跳过旧数据
```

**根本原因**:
- `os.getenv()` 返回的是**字符串**，不是布尔值
- `.env` 中设置 `FULL_UPDATE=false`
- Python 中 `not "false"` = `False`（非空字符串是真值）
- 导致跳过逻辑永远不执行

**修复方案**:
```python
# flomo2notion.py:385-386 (修复后)
full_update = os.getenv("FULL_UPDATE", "false").lower() == "true"
interval_day = int(os.getenv("UPDATE_INTERVAL_DAY", "7"))
```

**影响范围**:
- 所有用户使用非全量同步模式时
- 导致处理大量旧数据（2年前的数据也被处理）
- 同步时间从预期 2 分钟变成 208 分钟

**测试验证**:
```python
# 测试环境变量解析
os.environ["FULL_UPDATE"] = "false"
full_update = os.getenv("FULL_UPDATE", "false").lower() == "true"
assert full_update == False  # ✅ 通过
```

### 2. 时间字符串比较 BUG

**严重程度**: 🔴 高（导致智能更新失效）

**问题描述**:
```python
# flomo2notion.py:263 (修复前)
if notion_updated and flomo_updated <= notion_updated:
    # 只更新标题
```

**根本原因**:
- 直接用字符串比较时间
- `"2026-03-25 14:24:54" <= "2026-03-25T14:24:00.000+08:00"`
- 字符串比较：`' ' (空格, ASCII 32) < 'T' (ASCII 84)`
- 导致判断错误，误认为 Flomo 未更新

**修复方案**:
```python
# flomo2notion.py:265-295 (修复后)
# 解析为 datetime 对象
notion_dt = datetime.fromisoformat(notion_updated.replace('Z', '+00:00'))
flomo_dt = datetime.strptime(flomo_updated, '%Y-%m-%d %H:%M:%S')
beijing_tz = pytz.timezone('Asia/Shanghai')
flomo_dt = beijing_tz.localize(flomo_dt)

# 正确比较
should_full_update = flomo_dt > notion_dt

if notion_updated and flomo_updated and not should_full_update:
    # 只更新标题
```

**影响范围**:
- 所有需要智能更新的场景
- 导致内容已变化但只更新标题
- Notion 中的内容与 Flomo 不同步

**测试案例**:
```python
# 测试时间比较
flomo_time = '2026-03-25 14:24:54'
notion_time = '2026-03-25T14:24:00.000+08:00'

# 字符串比较（错误）
assert flomo_time <= notion_time  # ❌ True（错误）

# datetime 比较（正确）
flomo_dt = datetime.strptime(flomo_time, '%Y-%m-%d %H:%M:%S')
notion_dt = datetime.fromisoformat(notion_time.replace('+08:00', '+00:00'))
assert flomo_dt > notion_dt  # ✅ True（正确）
```

### 3. Notion Date 精度限制

**严重程度**: 🟡 中（API 限制，无法修复）

**问题描述**:
- Notion Date 属性只精确到**分钟**
- 秒数会被自动截断为 `:00`
- 无法存储 `2026-03-25 14:24:54`，只能存储 `2026-03-25 14:24:00`

**影响**:
- 如果 Flomo 更新时间是 `14:24:54`
- Notion 存储的时间是 `14:24:00`
- 时间差只有 54 秒，但无法判断是否真的有更新

**解决方案**:
```python
# 只比较到分钟级别，忽略秒数
notion_minutes = notion_dt.replace(second=0, microsecond=0)
flomo_minutes = flomo_dt.replace(second=0, microsecond=0)

# 如果分钟数相同，认为没有更新
should_full_update = flomo_minutes > notion_minutes
```

**遗留问题**:
- 如果在同一分钟内多次修改 Flomo memo，后续修改无法被检测
- 建议用户：修改后等待至少 1 分钟再同步

### 4. 缺少自动拉取数据

**严重程度**: 🔴 高（导致同步过期数据）

**问题描述**:
- `flomo2notion.py` 的 `sync_to_notion()` 方法
- 直接读取 `flomo_data.json`（可能是旧数据）
- 不会自动从 Flomo API 拉取最新数据
- 导致：用户在 Flomo 修改了 memo，但同步时用的是旧数据

**用户反馈**:
> "卧槽 以后同步之前肯定要重新啦取 flomo 数据的啊！之前难道没有拉吗？！？！"

**修复方案**:
```python
# flomo2notion.py:353-392 (修复后)
def sync_to_notion(self):
    # Step 0: 从 Flomo API 拉取最新数据
    print("="*70)
    print("📥 步骤0: 从 Flomo API 拉取最新数据")
    print("="*70)

    authorization = os.getenv("FLOMO_TOKEN")
    memo_list = []
    latest_updated_at = "0"

    while True:
        new_memo_list = self.flomo_api.get_memo_list(authorization, latest_updated_at)
        if not new_memo_list:
            break
        memo_list.extend(new_memo_list)
        latest_updated_at = str(int(time.mktime(...)))

    # 保存到 flomo_data.json
    import json
    flomo_data = {
        "memos": memo_list,
        "fetched_at": time.strftime("%Y-%m-%d %H:%M:%S", time.localtime())
    }

    with open('flomo_data.json', 'w', encoding='utf-8') as f:
        json.dump(flomo_data, f, ensure_ascii=False, indent=2)

    print(f"✅ 获取到 {len(memo_list)} 条 memo")
    print(f"✅ 数据已保存到 flomo_data.json")

    # Step 1: 建立映射...
```

**影响范围**:
- 所有同步操作
- 修复前：可能同步过期数据
- 修复后：始终同步最新数据

### 5. Mapping 数据缺失

**严重程度**: 🟡 中（导致无法更新特定页面）

**问题描述**:
- Notion 中存在页面（page_id: `32e66475-a5c8-817f-a521-e1a5ac181cc8`）
- slug: `MjI3OTU0MDgz`
- 但 `mapping.json` 中没有这个映射关系
- 导致同步时认为这是**新 memo**，尝试 `insert_memo()`
- Notion 返回错误（页面已存在）

**根本原因**:
- 可能是之前同步时创建了页面，但 mapping.json 没有保存
- 或者 mapping.json 被意外删除/重置

**修复方案**:
```python
# 手动添加映射
mapping['slug_to_page_id']['MjI3OTU0MDgz'] = '32e66475-a5c8-817f-a521-e1a5ac181cc8'

# 保存
with open('mapping.json', 'w', encoding='utf-8') as f:
    json.dump(mapping, f, indent=2, ensure_ascii=False)
```

**预防措施**:
- 建议在 `sync_to_notion()` 中添加 mapping 完整性检查
- 定期验证 Notion 页面与 mapping 的一致性

### 6. 标题过滤不完整

**严重程度**: 🟡 中（影响标题质量）

**问题描述**:
- 标题中包含 "关联自：https://..."
- 只删除了 "关联自"，没有删除后面的 URL

**错误示例**:
```
输入: "关联自：https://v.flomoapp.com/mine/?memo_id=XXX\n\n今天又想了很多..."
错误输出: "https://v.flomoapp..." (URL 还在)
正确输出: "今天又想了很多..."
```

**根本原因**:
```python
# utils.py:84-86 (修复前)
s = re.sub(r'^关联[自到][：:]\s*', '', s.strip())
# 只删除了 "关联自："，URL 还在
```

**修复方案**:
```python
# utils.py:84-86 (修复后)
s = re.sub(r'^关联[自到][：:]\s*https?://[^\n]*', '', s.strip())
# 删除整行："关联自：https://..."
```

**测试案例**:
```python
from utils import truncate_string

content = '''关联自：https://v.flomoapp.com/mine/?memo_id=MTExNzIwNTQ2

今天又产生一个点子，自然界其实也充满了各种矛盾斗争...'''

title = truncate_string(content)
assert title.startswith("今天又产生一个点子")  # ✅ 通过
assert "https://" not in title  # ✅ 通过
```

**影响范围**:
- 所有包含 "关联自" 的 memo
- 修复后批量处理了 29 个问题页面

## 📝 遗留问题

### 1. 同一分钟内的多次修改无法检测

**问题**:
- Flomo 在 `14:24:54` 修改
- Notion 存储为 `14:24:00`
- 如果用户在 `14:24:30` 再次修改，同步程序无法检测

**影响**: 低（极少见）

**建议方案**:
- 在 Flomo 中添加版本号字段
- 或使用内容 hash 判断是否变化

### 2. Mapping 完整性检查

**问题**:
- 没有 automated check 确保 mapping 与 Notion 一致
- 可能导致部分页面无法更新

**影响**: 中

**建议方案**:
- 添加 `check_mapping_integrity.py` 脚本
- 在 TODO.md 中记录

### 3. 错误处理不够友好

**问题**:
- API 调用失败时，错误信息是原始异常
- 用户难以理解

**影响**: 低

**建议方案**:
- 添加友好的错误提示
- 提供解决方案建议
